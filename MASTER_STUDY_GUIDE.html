This document acts as a "Textbook" for the ScreenAI project. It is designed for developers who need to understand every part of the system, from the database schema to the AI logic.

ScreenAI is a Recruitment Automation Platform.

  • Frontend: React.js (Client-side rendering, Component-based).
  • Backend: Django (API Server, Business Logic, Database).
  • AI Engine: Google Gemini (Resume parsing).
  • Database: SQLite (Local development default).

ðŸ“š Module 2: The Backend (Django)

The backend is organized into "Apps". Each app handles a specific domain.

2.1 Core App (appscreenai)

  • Purpose: Manages Users (Recruiters) and Global Analytics.
  • Key File: models.py - Employee model.
  • Key File: api_views.py - Handles dashboard calculations like Conversion Rate.

2.2 Jobs App (jobs)

  • Purpose: Manages Job Postings.
  • Key File: models.py - JobPosting.
    • Includes screening_questions (JSON) for custom forms.
    • Includes required_skills (JSON) for AI matching.

2.3 Candidates App (candidates)

  • Purpose: Manages Applicants and Resumes.
  • Key File: views.py - PreviewResumeView. This is the Autofill Endpoint that calls the AI.

ðŸ“š Module 3: The AI Engine (The "Brain")

Location: backend/screenai/services/resume_parser/parser.py

3.1 The Workflow

  1. Text Extraction: Uses pdfplumber to read PDF text.
  2. Prompt Engineering: Sends specific instructions to Gemini ("Extract specific JSON...").
  3. Smart Rotation: Rotates through multiple API keys if one fails.
  4. Fallback: Uses Regex if the AI fails completely.

ðŸ“š Module 4: The Frontend (React)

4.1 Key Pages

  • ApplyPage.js: The Candidate Form. Uses FormData to handle file uploads. Updates state dynamically on file drop.
  • DashboardPage.js: The Recruiter View. Uses recharts for visualization.

4.2 Services

api.js is the central bridge between Frontend and Backend using Axios.
